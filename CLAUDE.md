# CLAUDE.md — LynxPrompt VS Code

## Overview
The official VS Code extension for LynxPrompt that lets users browse, pull, diff, and push AI config files (AGENTS.md, CLAUDE.md, .cursor/rules, etc.) directly from the editor.

## Tech Stack
- TypeScript
- VS Code Extension API
- esbuild (bundler)
- Vitest (testing)
- ESLint

## Development
```bash
npm install
npm run compile      # esbuild bundle
npm run test         # vitest
npm run lint         # eslint
npm run watch        # esbuild watch mode
```
Press F5 in VS Code to launch the Extension Development Host.

## Architecture
- `src/extension.ts` — activation entrypoint
- `src/api.ts` — LynxPrompt API client
- `src/auth.ts` — authentication/token management
- `src/types.ts` — type definitions
- `src/commands/` — registered VS Code commands
- `src/views/` — tree view providers (sidebar)
- `src/utils/` — shared utilities
- `tests/` — test suite
- `media/` — icons and banner assets
- `esbuild.js` — build configuration
- `CHANGELOG.md` — version history
- `.vscodeignore` — files excluded from VSIX package

## Key Rules
- Never hardcode API tokens; use VS Code SecretStorage for credentials
- Published on VS Code Marketplace as `LynxPrompt.lynxprompt`
- Keep bundle size minimal via esbuild (no webpack)
- Supports LynxPrompt self-hosted instances (configurable API URL)
