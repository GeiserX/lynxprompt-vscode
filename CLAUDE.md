# CLAUDE.md — LynxPrompt VS Code

## Overview
The official VS Code extension for LynxPrompt that lets users browse, pull, diff, and push AI config files (AGENTS.md, CLAUDE.md, .cursor/rules, etc.) directly from the editor.

## Tech Stack
- TypeScript
- VS Code Extension API
- esbuild (bundler)
- Vitest (testing)
- ESLint
- OAuth device flow via LynxPrompt API + VS Code SecretStorage

## Development
```bash
npm install
npm run compile      # esbuild bundle
npm run test         # vitest
npm run lint         # eslint
npm run watch        # esbuild watch mode
npm run package      # create .vsix package
npm run publish      # publish to VS Code Marketplace
```
Press F5 in VS Code to launch the Extension Development Host.

## Architecture
- `src/extension.ts` — activation entrypoint, registers all commands
- `src/api.ts` — HTTP client for LynxPrompt API (list, get, create, update)
- `src/auth.ts` — OAuth device flow + token management via SecretStorage
- `src/types.ts` — shared TypeScript interfaces
- `src/commands/signIn.ts` — device-flow sign-in command
- `src/commands/signOut.ts` — clear stored credentials
- `src/commands/pull.ts` — download blueprint to workspace file
- `src/commands/push.ts` — upload local config to LynxPrompt cloud
- `src/commands/diff.ts` — side-by-side diff: local vs cloud version
- `src/commands/generate.ts` — invoke LynxPrompt wizard to create new config
- `src/utils/configDetector.ts` — glob-based detection of AI config files in workspace
- `src/utils/fileMapping.ts` — maps blueprint types to file paths (AGENTS_MD -> AGENTS.md)
- `src/views/blueprintTree.ts` — TreeDataProvider for "My Blueprints" sidebar
- `src/views/localFilesTree.ts` — TreeDataProvider for "Local Config Files" sidebar
- `tests/` — test suite
- `media/` — icons and banner assets
- `esbuild.js` — build configuration
- `CHANGELOG.md` — version history

| Command | ID | Description |

| Sign In | `lynxprompt.signIn` | OAuth device-flow authentication |
| Sign Out | `lynxprompt.signOut` | Clear stored credentials |
| Pull Blueprint | `lynxprompt.pullBlueprint` | Download cloud blueprint to workspace |
| Push Config | `lynxprompt.pushConfig` | Upload local AI config to LynxPrompt |

## Key Rules
- Never hardcode API tokens; use VS Code SecretStorage for credentials
- Published on VS Code Marketplace as `LynxPrompt.lynxprompt`
- Keep bundle size minimal via esbuild (no webpack)
- Supports LynxPrompt self-hosted instances (configurable API URL)
- All API calls go through `src/api.ts` — never call `fetch` directly from commands
- Tree views refresh via event emitters, not polling
- The extension activates on workspace file detection (see `activationEvents` in package.json)
- Bundle output is a single file at `dist/extension.js` via esbuild

## Configuration

The extension reads these VS Code settings:

- `lynxprompt.apiUrl` — API base URL (default: `https://lynxprompt.com`)
- `lynxprompt.autoDetectConfigFiles` — auto-detect AI configs in workspace
- `lynxprompt.watchFileChanges` — watch linked files for changes
- `lynxprompt.showStatusBar` — show connection status in status bar
