# AGENTS.md — lynxprompt-vscode

> VS Code extension for [LynxPrompt](https://lynxprompt.com) — browse, pull, push, and manage AI configuration files directly from the editor.

## Project Overview

This is the official VS Code extension for LynxPrompt, the open-source platform for managing AI IDE configuration files (AGENTS.md, CLAUDE.md, .cursorrules, copilot-instructions.md, .windsurfrules, etc.) across 40+ AI coding tools.

The extension provides a sidebar panel with two tree views:
- **My Blueprints** — lists the user's cloud blueprints from LynxPrompt
- **Local Config Files** — auto-detects AI config files in the workspace

## Tech Stack

- **Language**: TypeScript
- **Runtime**: VS Code Extension Host (Node.js)
- **Bundler**: esbuild (single-file output to `dist/extension.js`)
- **Auth**: OAuth device flow via LynxPrompt API + VS Code SecretStorage
- **API**: REST calls to `lynxprompt.com/api/v1/` (configurable via `lynxprompt.apiUrl` setting)

## Architecture

```
src/
├── extension.ts           # Activation entry point, registers all commands
├── api.ts                 # HTTP client for LynxPrompt API (list, get, create, update)
├── auth.ts                # OAuth device flow + token management via SecretStorage
├── types.ts               # Shared TypeScript interfaces
├── commands/
│   ├── signIn.ts          # Device-flow sign-in command
│   ├── signOut.ts         # Clear stored credentials
│   ├── pull.ts            # Download blueprint → workspace file
│   ├── push.ts            # Upload local config → LynxPrompt cloud
│   ├── diff.ts            # Side-by-side diff: local vs cloud version
│   └── generate.ts        # Invoke LynxPrompt wizard to create new config
├── utils/
│   ├── configDetector.ts  # Glob-based detection of AI config files in workspace
│   └── fileMapping.ts     # Maps blueprint types ↔ file paths (AGENTS_MD → AGENTS.md)
└── views/
    ├── blueprintTree.ts   # TreeDataProvider for "My Blueprints" sidebar
    └── localFilesTree.ts  # TreeDataProvider for "Local Config Files" sidebar
```

## Key Commands

| Command | ID | Description |
|---|---|---|
| Sign In | `lynxprompt.signIn` | OAuth device-flow authentication |
| Sign Out | `lynxprompt.signOut` | Clear stored credentials |
| Pull Blueprint | `lynxprompt.pullBlueprint` | Download cloud blueprint to workspace |
| Push Config | `lynxprompt.pushConfig` | Upload local AI config to LynxPrompt |
| Compare with Cloud | `lynxprompt.diffBlueprint` | VS Code diff editor: local vs remote |
| Generate Config | `lynxprompt.generateConfig` | Open LynxPrompt wizard in browser |
| Convert Format | `lynxprompt.convertFormat` | Convert between AI config formats |

## Development

```bash
npm install          # Install dependencies
npm run watch        # Dev mode with auto-rebuild
npm run build        # Production build
npm run package      # Create .vsix package
npm run publish      # Publish to VS Code Marketplace
```

Press F5 in VS Code to launch the Extension Development Host for debugging.

## Configuration

The extension reads these VS Code settings:

- `lynxprompt.apiUrl` — API base URL (default: `https://lynxprompt.com`)
- `lynxprompt.autoDetectConfigFiles` — auto-detect AI configs in workspace
- `lynxprompt.watchFileChanges` — watch linked files for changes
- `lynxprompt.showStatusBar` — show connection status in status bar

## Conventions

- All API calls go through `src/api.ts` — never call `fetch` directly from commands
- Auth tokens are stored in VS Code's SecretStorage (never in plain settings)
- Tree views refresh via event emitters, not polling
- The extension activates on workspace file detection (see `activationEvents` in package.json)
- Bundle output is a single file at `dist/extension.js` via esbuild
